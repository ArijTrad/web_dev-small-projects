📦 Box-Sizing: Content-Box vs Border-Box

💡 The Problem: By default, CSS uses content-box. When you set width: 300px, that's JUST the content. Padding and border get ADDED on top, making the box wider than you wanted!

❌ PROBLEM: Default Content-Box (BROKEN)

CSS says: width: 300px; padding: 20px; border: 5px;

I am 300px wide... or am I? 🤔
📏 I wanted: 300px total
🚫 I actually am: 350px total
(300 + 20左 + 20右 + 5左 + 5右 = 350px)

This BREAKS layouts because the box doesn't fit where you expect it to!

✅ FIX: Border-Box (WORKING)

CSS says: width: 300px; padding: 20px; border: 5px; box-sizing: border-box;

I am ACTUALLY 300px wide! 🎯
📏 I wanted: 300px total
✅ I actually am: 300px total
(padding and border are INSIDE the 300px)

The box fits EXACTLY where you expect it to!

🌍 Real World Example: 3 Columns Layout

❌ WITHOUT border-box (Broken Layout)

Column 1
(Should be 33.3% of parent)
Column 2
(Should be 33.3% of parent)
Column 3
(Should be 33.3% of parent)

❌ They don't fit! They wrap to next line because padding+border makes each >33.3%

✅ WITH border-box (Working Layout)

Column 1
(Exactly 33.3% of parent)
Column 2
(Exactly 33.3% of parent)
Column 3
(Exactly 33.3% of parent)

✅ Perfect! All 3 fit side by side because padding+border are INSIDE the width

🔧 THE SOLUTION (put this at the top of your CSS):

* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}

The * selector applies this to EVERY element on your page, saving you from layout headaches forever!
